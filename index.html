AI Nurse Robot Platform

A connected system where caregivers manage senior care info through a portal, the robot screen guides the senior with calm prompts, and a Python backend coordinates reminders + real-time alerts (with future support for YOLO + robot movement).

Quick Summary

  • Nurses/kids (caregivers) can log in and enter/update senior info (meds, contacts, conditions, notes).
  • Robot screen UI (running on a MacBook later) shows simple calm prompts, reminders, and call buttons.
  • Backend “brain” connects everything + sends alerts/events (med due, fall detected, etc.).
  • Hardware/robot movement + YOLO integration come later (but architecture supports it).
Portal Robot UI Python Brain Realtime Events

Care Portal (web)

Core actions
  • Login (nurse/child)
  • Create a senior profile
Enter/edit
  • Meds + schedules
  • Emergency contacts
  • Notes / symptoms / problems
View
  • Event history (missed meds, alerts, future fall alerts)
  • Adherence (taken/missed)

Robot UI (web or local UI on robot)

  • Big text + simple buttons
  • “Next medication” + “Taken / Snooze”
  • “Call nurse / Call child”
  • Event banners (ex: “It’s time for your medication”)
Note: This UI will run on the robot’s MacBook screen later. For now, it’s a UI that connects to the backend.

Backend Brain (Python)

  • Stores all data in database
  • Sends real-time events to UIs
  • Scheduling system for meds/check-ins
Later
  • YOLO fall detection events
  • Robot movement commands
  • AI assistant reasoning

Tech / Tools I’ll likely use

  • Backend: Python (FastAPI)
  • Database: Postgres (or SQLite for early local testing)
  • Realtime events: WebSockets
  • Portal UI: simple web frontend (Next.js or basic HTML pages early)
  • Robot UI: web UI (kiosk-style) or simple local UI
  • AI later: OpenAI API (tool-using “brain” that reads/writes the database)
  • Version control: GitHub monorepo (apps + services in one place)
Goal: keep the system modular — portal + robot UI share the same backend and database so everything stays interconnected.
